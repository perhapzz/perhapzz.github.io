Linux 学习#

Linux 版本#

Linux 分为内核版本和发行版本

（1）内核版本#

内核版本分为三个部分：主版本号、次版本号、末版本号（次版本号是奇数为开发版，偶数为稳定版）

（2）发行版本#

常见的发行版本有5种：

RedHat Enterprise：软件经过专业测试，非常稳定，技术支持需要付费

Fedora：通过组建社区提供支持，没有专业测试

CentOS：基于 RedHat 源码进行编译，去掉标识，可免费使用

Debian 和 Ubuntu：定制了界面

Linux 优势#

安全、开源、性能高

常见目录#

目录名 目录作用
/bin 存放系统命令的目录，普通用户和root用户都可以执行，且单用户模式也可以执行
/usr/bin 存放系统命令的，是在后期安装的软件的运行脚本，普通用户和root用户都可以执行，这些命令与系统启动无关，无法在单用户模式下执行
/usr/sbin 存放根根文件系统无需的命令。例如多数服务程序，只有root用户可以使用
/boot 系统启动目录，保存系统启动相关的文件，如内核文件和启动引导文件
/dev 设备文件存放目录。在linux中，设备也是以文件的形式存在
/etc 配置文件存放目录。所有采用默认安装（rpm安装）的服务的配置文件都会存放在这个目录
/home 普通用户的宿主目录，每个普通用户都要有一个默认的登录位置，如用户user1的宿主目录就是/home/user1
/root root用户的宿主目录
/lib 系统调用的函数库保存位置
/opt 第三方安装的软件保存位置。但更多是放置到/usr/local目录当中
/media 挂载目录。建议用来挂载媒体设备。如软盘，光盘
/mnt 挂载目录。建议挂载额外设备，如U盘、移动硬盘等
/misc 挂载目录。建议挂载NFS服务的共享目录。
/proc 虚拟文件系统，是保存在内存中的。主要保存系统的内核，进程和外部设备状态和网络状态等。
/sys 虚拟文件系统，保存在内存中。主要保存内核相关信息。
/srv 服务数据目录。一些系统服务启动之后，可以在这个目录中保存所需要的数据
/tmp 系统存放临时文件的目录。建议不要保存重要数据且在每次开机都清空。
/var 动态数据保存位置。主要保存缓存、日志等。

磁盘管理#

基本操作#

  1. 目录信息查看：

    1. 查看当前目录：pwd
    2. 查看目录信息：ls

  3. 目录路径切换：cd

展示目录文件：ls#

  1. ls -a (all)：展示所有文件，含隐藏
  2. ls -ltr (long time reverse)：展示详细信息，按照时间排序
  3. ls 目录名：展示目录文件

切换工作目录：cd#

  1. cd 相对路径
  2. cd 绝对路径

创建删除目录：mkdir/rmdir#

  1. mkdir -p：递归创建目录

文件管理#

文件拷贝：cp#

  1. cp-copy file：复制文件或目录 (-r)

文件移动/改名：mv#

  1. mv source directory：把 source 移动到 dir
  2. mv source dest：把 source 改名为 dest

文件删除：rm#

  1. rm：删除普通文件
  2. rm -rf：直接删除文件&文件夹

读取文件：cat#

  1. cat *.txt：输出到屏幕
  2. cat a.txt b.txt：输出到文件流

文件开头/结尾：head/tail#

  1. head/tail -n num XXX：查看 XXX 文件的开头/结尾的 num 行（若不写默认 num=10）

文件差异比较：diff#

  1. diff A B：对比 A 和 B 有何不同，进阶用法查看 diff -help

文件压缩与解压：tar/zip#

  1. tar -zcvf/tar -zxvf：打包/解压 tar 格式文件
  2. zip/unzip：打包/解压 zip 格式文件

注：tar 和 zip 都是文件压缩格式，但是它们的区别在于：tar 是一种归档文件，可以将多个文件或目录打包在一起，但不会进行压缩。而 zip 则是一种压缩文件格式，可以将多个文件或目录打包在一起，并进行压缩。zip 格式的压缩比较高，但是 tar 的速度比 zip 快。此外，tar 和 zip 的使用场景也不同，tar 通常用于 Unix 和类 Unix 系统上的压缩打包工具，而 zip 则主要被使用在 Windows 系统下。

帮助命令：man#

使用场景：忘记某个命令参数的含义了，可以用来查命令的详细文档、配置文件的详细说明

  1. man ${command}：查询命令的详细信息
  2. man 5 ${filename}：查询文件的详细信息

删除用户密码：passwd#

  2. passwd work：修改其他用户密码（只有拥有 root 权限的用户可以）

文件与目录权限管理#

修改权限：chmod#

  1. chmod 750 filename：修改权限
  2. chmod -R 750 dirname：递归修改文件权限

权限解释：

  1. root 用户拥有系统最高权限，慎用

  2. 权限划分 -rwxrwxrwx-

    1. 前三位 U user 归属人
    2. 中三位 G group 归属组
    3. 后三位 O 其他人

  4. 数字权限：r = 4（读）, w = 2（写）, x = 1（执行）

修改归属：chown#

  1. chown user:group filename：修改归属人以及归属组
  2. chown -R user:group dirname：递归修改改目录下全部文件

文件修改：sed#

  1. sed -i 's/aaa/bbb/g' filename：替换文件内容，这段语句将 filename 文件中所有的 aaa 替换成 bbb

    1. s 代表替换
    2. g 代表全局执行

  3. sed -i 1,3d filename：删除文件内容，这段语句删除文件前三行内容

    1. 1 代表起始
    2. 3 代表结束
    3. d 代表删除

  5. sed -i 5d filename：删除文件内容，这段豫剧删除文件第五行内容

文本处理命令：awk#

  1. awk '{print $2}' filename：以空格为分隔符，输出第二列
  2. awk '{print $NF}' filename：以空格为分隔符，输出最后一列
  3. awk -F 'a' '{print $2}' filename：以 a 为分隔符，输出第二列
  4. awk '$2 ~ /th/ {print $2,$4}' filename：第二列包含 th 时，输出第2、4列

命令模式#

  1. 解释：打开 vim 之后默认进入的模式，进入到 vim 最强大的命令模式

  2. 切换方法：可从任意模式通过 ESC 键切换到命令模式

  3. 命令模式中常用的命令

      1. :w：保存
      2. :wq：保存退出
      3. :x：保存退出
      4. :q：退出（已保存）
      5. :q!：不保存强制退出
    1. 移动光标命令
      1. h, j, k, l：光标移动：h左，j下，k上，l右
      2. ctrl+f：上一页
      3. ctrl+b：下一页
      4. w, e：跳到单词的后面
      5. b, B：跳到单词的前面
      6. gg：跳到第一行开始
      7. G：跳到最后一行开始
    1. 复制粘贴命令
      1. dd：剪切一行
      2. 3dd：剪切3行，同理 4dd, 5dd
      3. dw：删除一个单词
      4. x：删除光标所在字符
      5. yy：复制一行
      6. 3yy：复制3行，与3dd同理
      7. p：粘贴内容到光标的下一行
      8. P：粘贴内容到光标的上一行
插入模式#

  1. 解释：类似 windows 记事本

  2. 切换方法：只能从命令模式通过（i, a, I, A）等命令进入插入模式

    1. i：插入到光标前面
    2. I：插入到光标所在行最前面
    3. a：插入到光标后面
    4. A：插入到光标所在行最后面

    6. O：在光标所在行上面新开一行

  1. 解释：使用光标选取一片区域的文本内容，之后可通过命令来操作

Git 学习#

分支管理策略#

master 分支#

git 主分支的名字，它是自动建立的，版本库初始化后，默认就是在主分支进行开发。

release 分支#

release 分支是指发布正式版本之前（即合并到 master 分支之前），我们可能需要有一个预发布版本测试从 develop 创建的分支。预发布结束后，必须合并进 develop 和 master 分支。

develop 分支#

日常开发分支，当 develop 分支中的源代码达到稳定点并准备发布时，所有更改都应以某种方式合并回 master。

feature 分支#

功能分支，为了开发某种特定功能，从 develop 上分出来的。开发完后要合并进 develop。

hotfix 分支#

hotfix 从 master 上 checkout 出来，必须合并回 master，如果存在待发布的 release 分支，则还需要合并到 release 分支，否则就需要合并到 develop 分支。如果 develop 立即进行工作需要此错误的修正，并且不能等待发行分支完成，则也可以安全地将错误修正合并到 develop 分支。

git add 和 git rm –cached #

  1. git add <file>：将文件添加到暂存区
  2. git add .：将所有文件添加到暂存区
  3. git rm --cached <file>：撤销文件添加到暂存区的修改

  2. git log --oneline --graph：以图表的形式看到当前分支的所有提交

git fetch#

  1. git fetch <远程主机名> <远程分支名>:<本地分支名>：从远处仓库的分支拉取代码到本地仓库来

git fetch / git remote / git pull#

git fetch 和 git remote update origin 都是用来从远程仓库更新本地仓库的代码，但是它们之间有一些区别。

git fetch 命令会将远程仓库的代码下载到本地仓库，但是不会自动合并到本地分支。这意味着你需要手动合并代码。

而 git remote update origin 命令则会将远程仓库的代码下载到本地仓库，并且自动合并到本地分支。

所以，如果你想要手动合并代码，可以使用 git fetch 命令；如果你想要自动合并代码，可以使用 git remote update origin 命令。

git pull` = `git fetch` + `git merge FETCH_HEAD

git push#

  1. git push origin master:master：将本地仓库的 master 分支推到远程仓库的 master 分支分支上
  2. git push：默认将本地当前分支推到远程相同名字分支上
