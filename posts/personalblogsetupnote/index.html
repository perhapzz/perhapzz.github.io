个人博客搭建笔记

记用hugo搭建个人博客的全过程
December 12, 2022

基础版：#

主要参考B站CodeSheep视频

1. 安装 Hugo#

# 安装 hugo
$ brew install hugo

# 测试是否安装成功
$ hugo version

2. 用 Hugo 生成博客#

$ hugo new site Blog

3. 下载并设置主题#

我选择的主题是PaperMod，其他主题可以在Hugo Themes中查找

$ cd Blog/
$ git clone https://github.com/adityatelange/hugo-PaperMod themes/PaperMod --depth=1

4. 本地启动个人博客#

$ hugo server -t PaperMod --buildDrafts

顺利启动了，访问http://localhost:1313/可以看到界面：

5. 写一篇文章#

$ hugo new post/blog.md

这样我第一篇文章就生成了，接下来直接在修改blog.md即可～

6. 将个人博客部署到远端服务器#

首先新建一个github仓库，命名一定要{github id}.github.io 

# 生成最终页面
$ hugo --theme=PaperMod --baseUrl="https://perhapzz.github.io" --buildDrafts

自动帮我在public中转成html了

$ cd public

$ git init
# Initialized empty Git repository in /Users/hzpan/Blog/public/.git/
$ git add .
$ git commit -m "Initial commit"

$ git remote add origin https://github.com/perhapzz/perhapzz.github.io.git
$ git push -u origin master

项目就提交到master分支上去了，接下来访问https://perhapzz.github.io/，就可以看到我的个人博客啦～

7. 添加或修改文档后更新#

# 在 Blog 目录下运行指令，更新 public
$ hugo --theme=PaperMod --baseUrl="https://perhapzz.github.io" --buildDrafts

# 在 public 目录下运行执行，push到仓库
$ git add.
$ git commit -m "commit information"
$ git push -u origin master

进阶版：#

1. hugo配置#

参考：Getting Started With Hugo | FREE COURSE

我将在后面使用 yml/yaml 格式，因为 yml 相较于 toml 更容易阅读。（YML到TOML的转换器）

也可以在hugo生成博客的时候就以 yml 格式生成。

# Use -f to select yml format
$ hugo new site <name of site> -f yml

我这边直接重建

# 直接进入博客文件夹把主题 clone 下来
$ cd <name of site>
$ git clone https://github.com/adityatelange/hugo-PaperMod themes/PaperMod --depth=1

这里有个比较好的方法可以优化上面的指令，就是直接进入config.yml改

baseURL: https://perhapzz.github.io
theme: "PaperMod"

这样在本地启动博客的指令为

$ hugo server
# -D –buildDrafts 在生成静态网站，或预览网站时，草稿内容也会被展示出来，也可以不加

接下来生成一篇博客，可以以archetypes文件夹里的文件进行模板化生成，默认使用archetypes/default.md格式生成一篇文档

$ hugo new --kind default <name>

然后执行hugo指令生成最终页面

# 因为前面在config.yml配置好theme和baseUrl，所以可以直接hugo
$ hugo

然后进入public目录提交到仓库就行了

$ cd public
$ git add.
$ git commit -m "commit information"
$ git push -u origin master
