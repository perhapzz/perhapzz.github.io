青训营大项目复盘#

整个大项目也差不多做完了，我在其中主要参与的是 composite 部分和后续接手的 video，主要开发的接口包括获取视频流、点赞评论操作以及发布视频（接盘）这几个部分，涉及到的技术框架包括 Kitex、Herz、Gorm、Jeager、Etcd、Thrift 还有阿里的对象存储数据库 OSS。

其中 Kitex[kaɪt'eks] 用作微服务 RPC 框架，Hertz 用作微服务 HTTP 框架，Gorm 用于操作数据库，其中也使用了 gorm-gen 用于代码生成，Jeager 用作 gorm 的链路追踪，Etcd 用于服务注册和发现，Thrift 用于接口文档的编写。

我最终提交的代码大约 3-4w 行左右，最终算是顺利完成我本职工作内的任务。

但是我对于自己还是存在一些不满意的地方，比如因为大伙时间不够充裕，最后没有加上 Redis 作为缓存、 RabbitMQ 作为消息队列以及通过并发操作提升整体性能。

借口总是很多，还是应该好好反思和整理收获：

反思#

1. 时间#

这次项目我个人感觉没有做好合理的时间规划，整体进展速度没有那么理想，主要是我自己的能力也不太行。希望以后能够做好时间规划和阶段性总结，努力做到更快更好。

2. 仓库管理#

我觉得这次项目的仓库管理做得并不是很理想，同时也感叹一个漂亮的代码仓库来之不易。

  1. commit 规范很重要，可以使得整个项目的路线清晰可见，会让人有种项目慢慢成长起来的成就感。
  2. 项目初步开发完毕后，后续增添功能、修改 bug 都最好以 issue 和 pr 的形式完成，这样的工作方式可以给人一种事事有回应的畅快感受，同时 pr 也能给学有余力的同学 review 的机会，可以带动大家一起成长。

收获#

1. 框架设计#

下图是我们队长 王冠骅 同学设计的架构图，我觉得很精彩，除了 chat 部分我不知道最后实现了没，其他部分我觉得通过大家的努力最后都顺利完成了。

下次再做，我说不定还是会按照这个框架设计。但是在保证功能顺利完成的前提下（下次动作应该能快点儿了吧），我希望能够在性能上有所提升，加入 Redis 和一些并发操作，并且加入消息队列等等。

2. 开发体验#

参考这篇博文 给想转Go或者Go进阶同学的一些建议， 一个理想的开发流程大概如下：

  1. 通读一遍需求文档和原型图
  2. 梳理业务逻辑，进行抽象，明确有多少个功能需求要开发
  3. 根据功能需求创建数据库，创建表，添加字段，设置合适的字段类型，长度，主外键等
  4. 考虑业务场景，创建索引…
  5. 开始疯狂的CRUD…
  6. 开始疯狂的加Cache…
  7. 开发疯狂的给客户端提供数据接口…
  8. 持续迭代：根据业务增长做负载均衡、分库分表、读写分离….

这是我在做大项目前看到的文章，现在回头又翻出来再看一遍，感触颇多，遗憾颇多。从一开始的云里雾里，再到现在看到每一条都能回想起当时做这些内容的场景（不过加 Cache 和持续迭代等部分我还没完全体验到），我为自己确确实实体验到了一些开发的流程感到开心吧，也确确实实学到了很多东西。

最后，非常开心遇到了靠谱的队友们，与队友们聊天交流非常非常愉快，非常感恩有一次这样的体验～
